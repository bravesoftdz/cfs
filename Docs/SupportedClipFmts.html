CFS
Supported Clipboard Formats

CF_BITMAP

Standard Format: CF_BITMAP (2)

A handle to a bitmap (HBITMAP). Stored on clipboard as a GDI handle.

CFS supports using:

  • UBmpViewer.TBmpViewer (primary viewer)

CF_DIB

Standard Format: CF_DIB (8)

A device independent bitmap stored in global memory as a BITMAPINFO structure followed by the bitmap's bits.

CFS supports using:

  • UDIBViewer.TDIBViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

CF_ENHMETAFILE

Standard Format: CF_ENHMETAFILE (14)

A handle to an enhanced metafile (HENHMETAFILE). Stored on clipboard as a GDI handle.

CFS supports using:

  • UMetafileViewer.TMetafileViewer (primary viewer)

CF_HDROP

Standard Format: CF_HDROP (15)

A handle of type HDROP that identifies a list of files in global memory. Read the file list using the DragQueryFile function.

CFS supports using:

  • UHDROPViewer.THDROPViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

CF_LOCALE

Standard Format: CF_LOCALE (16)

The data is a handle to the locale identifier associated with text in the clipboard. It is stored in global memory as a pointer to a LCID structure.

CFS supports using:

  • ULocaleViewer.TLocaleViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

CF_METAFILEPICT

Standard Format: CF_METAFILEPICT (3)

A handle to a metafile picture format as defined by the METAFILEPICT structure.

CFS supports using:

  • UMetafileViewer.TMetafileViewer (primary viewer)

CF_OEMTEXT

Standard Format: CF_OEMTEXT (7)

Text format containing characters in the OEM character set. Each line ends with CRLF.

Stored in global memory as a zero terminated ANSI string.

CFS supports using:

  • UTextViewer.TOEMTextViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

CF_TEXT

Standard Format: CF_TEXT (1)

Text format containing ANSI characters. Each line ends with CRLF.

Stored in global memory as a zero terminated ANSI string.

CFS supports using:

  • UTextViewer.TAnsiTextViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

CF_UNICODETEXT

Standard Format: CF_UNICODETEXT (13)

Text format containing Unicode characters. Each line ends with WideChar CRLF.

Stored in global memory as a WideChar zero terminated Unicode string.

CFS supports using:

  • UTextViewer.TUnicodeTextViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

FileGroupDescriptor

Custom format: CF_FILEGROUPDESCRIPTORA

Provides a description of a group of files.

Stored in global memory as an ANSI FILEGROUPDESCRIPTORA structure which comprises a count of the number of files in the group followed by the required number of FILEDESCRIPTORA structures.

CFS supports using:

  • UFileGroupDescViewer.TFileGroupDescViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

FileGroupDescriptorW

Custom format: CF_FILEGROUPDESCRIPTORW

Provides a description of a group of files.

Stored in global memory as a Unicode FILEGROUPDESCRIPTORW structure which comprises a count of the number of files in the group followed by the required number of FILEDESCRIPTORW structures.

CFS supports using:

  • UFileGroupDescViewer.TFileGroupDescViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

FileName

Custom format: CF_FILENAMEA

Defines a file name as ANSI text. Default code page is assumed.

Stored in global memory as a zero terminated ANSI string.

CFS supports using:

  • UFileNameViewer.TAnsiFileNameViewer (primary viewer)
  • UTextViewer.TAnsiTextViewer
  • UBinaryViewer.TBinaryViewer

FileNameW

Custom format: CF_FILENAMEW

Defines a file name as Unicode text.

Stored in global memory as a WideChar zero terminated Unicode string.

CFS supports using:

  • UFileNameViewer.TUnicodeFileNameViewer (primary viewer)
  • UTextViewer.TUnicodeTextViewer
  • UBinaryViewer.TBinaryViewer

HTML Format

Custom format: CF_HTML

Defines a fragment of raw HTML text and its context. The fragment comprises a header stored as ASCII text followed by content as UTF-8 text. Since UTF-8 format and ASCII format are indentical for character codes up to 127, the data is treaded as UTF-8.

Stored in global memory as zero terminated UTF-8 text.

CFS supports using:

  • UHTMLClipViewer.THTMLClipViewer (primary viewer)
  • UTextViewer.TUTF8TextViewer
  • UBinaryViewer.TBinaryViewer

HTML (HyperText Markup Language)

Custom format: CF_HYPERTEXT

Contains a complete HTML document. Appears to be stored as ANSI text (default code page assumed), even if CONTENT-TYPE meta tag header says UTF-8. This code makes assumption of use of default ANSI code page.

Stored in global memory as zero terminated ANSI (see above).

CFS supports using:

  • UHTMLDocViewer.THyperTextViewer (primary viewer)
  • UTextViewer.TAnsiTextViewer
  • UBinaryViewer.TBinaryViewer

Link Source Descriptor

Custom format: CF_LINKSRCDESCRIPTOR.

Provides user interface information during data transfer operations. See also "Object Descriptor".

Stores required data in global memory as a OBJECTDESCRIPTOR structure.

CFS supports using:

  • UObjDescViewer.TObjDescViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

Object Descriptor

Custom format: CF_OBJECTDESCRIPTOR.

Provides user interface information during data transfer operations. See also "Link Source Descriptor".

Stores required data in global memory as a OBJECTDESCRIPTOR structure.

CFS supports using:

  • UObjDescViewer.TObjDescViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

Rich Text Format

Custom Format: CF_RTF

Stores rich text code as zero terminated ASCII text in global memory.

CFS supports using:

  • URTFViewer.TRTFViewer (primary viewer)
  • UTextViewer.TASCIITextViewer
  • UBinaryViewer.TBinaryViewer

Rich Text Format Without Objects

Custom Format: CF_RTFNOOBJS

As CF_RTF but code does not include any embedded objects. Stores rich text code as zero terminated ASCII text in global memory.

CFS supports using:

  • URTFViewer.TRTFViewer (primary viewer)
  • UTextViewer.TASCIITextViewer
  • UBinaryViewer.TBinaryViewer

RTF As Text

Custom format: CF_RTFASTEXT

Identical format to CF_RTF (zero terminated ASCII text in global memory) but designed for displaying RTF code instead of formatted text.

CFS supports using:

  • UTextViewer.TASCIITextViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

Shell IDList Array

Custom format: CF_IDLIST

This format is used when transferring the locations of one or more existing namespace objects. It is used in much the same way as CF_HDROP, but it contains PIDLs instead of file system paths.

The data is stored in global memory as a CIDA structure along with a number of ITEMIDLIST structures. Full details are on MSDN under the CFSTR_SHELLIDLIST constant.

CFS supports using:

  • UIDListViewer.TIDListViewer (primary viewer)
  • UBinaryViewer.TBinaryViewer

text/html

Custom format: CF_MIME_HTML

Contains a fragment of an HTML document, including all necessary opening and closing tags.

Stored in global memory as WideChar zero terminated Unicode text.

CFS supports using:

  • UHTMLDocViewer.TMIMEHTMLViewer (primary viewer)
  • UTextViewer.TUnicodeTextViewer
  • UBinaryViewer.TBinaryViewer

text/_moz_htmlcontext

Custom format: CF_MIME_MOZHTMLCONTEXT

Contains context of HTML currently on clipboard in text/html format. This context is the HTML of the branch of the DOM containing HTML in text/html. The context comprises only HTML tags: there is no text.

Stored in global memory as WideChar zero terminated Unicode text.

CFS supports using:

  • UTextViewer.TUnicodeTextViewer
  • UBinaryViewer.TBinaryViewer

UniformResourceLocator

Custom format: CF_INETURLA

Defines a URL as ASCII text. ASCII is assumed since properly URL encoded URLs should be ASCII.

Stored in global memory as a zero terminated ASCII string.

CFS supports using:

  • UURLViewer.TAnsiURLViewer (primary viewer)
  • UTextViewer.TASCIITextViewer
  • UBinaryViewer.TBinaryViewer

UniformResourceLocatorW

Custom format: CF_INETURLW

Defines a URL as Unicode text.

Stored in global memory as a WideChar zero terminated Unicode string.

CFS supports using:

  • UURLViewer.TUnicodeURLViewer (primary viewer)
  • UTextViewer.TUnicodeTextViewer
  • UBinaryViewer.TBinaryViewer
